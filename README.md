# 📝 PERN Todo Application

A full-stack CRUD (Create, Read, Update, Delete) application built using the PERN stack (PostgreSQL, Express, React, and Node.js). This project allows users to manage their daily tasks through a clean, responsive web interface.

## 🚀 Live Demo: https://client-3836.onrender.com

### ✨ Features

- Create: Add new tasks to your list.
- Read: Fetch and display all tasks from the PostgreSQL database in real-time.
- Update: Edit existing task descriptions using a Bootstrap modal.
- Delete: Remove tasks permanently from the list.
- Environment Configuration: Dynamic API URL management using environment variables.

## 🛠️ Tech Stack

### ⚛️ Frontend

- React.js: Functional components and Hooks (useState, useEffect).
- Bootstrap: Responsive UI and modal components.
- Fetch API: For making asynchronous requests to the backend.

### 🖥 Backend

- Node.js & Express: RESTful API development.
- PostgreSQL: Relational database for persistent storage.
- CORS: Cross-Origin Resource Sharing for secure communication between frontend and backend.

### 🗄️ Database

- PostgreSQL – Primary relational database used for storing todo tasks.
- SQL Queries – Custom scripts used to define the schema and manage data persistence.
- Connection Pooling – Managed via pg (node-postgres) for efficient database communication.

### 🔧 Other Tools & DevOps

- REST API – Architectural style used for communication between client and server.
- Render – Cloud platform used for hosting both the frontend and the backend.
- Postman – Used for testing API endpoints during the development phase.
